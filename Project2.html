Create an EC2 instance + deploy Jenkins with Terraform

main.tf :
Ce fichier principal de Terraform définit les ressources nécessaires pour créer une instance EC2 sur AWS et y déployer Jenkins. Il inclut les configurations pour la sécurité, le réseau, et les spécifications de l'instance.

variables.tf :
Ce fichier contient toutes les variables nécessaires à la configuration du déploiement, telles que les identifiants AWS, les informations sur le réseau, et les paramètres de Jenkins. Il permet de rendre le script plus modulaire et réutilisable.

outputs.tf :
Ce fichier définit les sorties de Terraform, telles que les adresses IP ou les URL nécessaires pour accéder à Jenkins une fois déployé. Ces sorties sont utiles pour obtenir rapidement les informations de connexion après le déploiement.

security_groups.tf :
Ce fichier configure les groupes de sécurité AWS pour l'instance EC2, en définissant les règles de pare-feu nécessaires pour permettre l'accès à Jenkins et autres services associés.

ec2_instance.tf :
Ce fichier spécifique configure les paramètres détaillés de l'instance EC2, y compris le type d'instance, le volume de stockage, et les balises pour l'organisation des ressources.

jenkins_install.sh :
Un script shell pour automatiser l'installation de Jenkins sur l'instance EC2. Il télécharge Jenkins, configure les paramètres initiaux, et assure que le service est démarré correctement.

Ces fichiers, lorsqu'ils sont utilisés ensemble, permettent de déployer rapidement et de manière fiable une instance Jenkins sur AWS en utilisant Terraform pour l'infrastructure et des scripts shell pour la configuration initiale de Jenkins. Lien GitHub

Write Terraform files for EKS and deploy Jenkins pipeline

main.tf :
Ce fichier principal de Terraform configure toutes les ressources nécessaires pour mettre en place un cluster EKS sur AWS. Il orchestre la création du cluster EKS, des groupes de nœuds et des ressources associées.

variables.tf :
Ce fichier contient les définitions de variables utilisées dans les configurations Terraform. Des variables telles que la région AWS, le nom du cluster et les types d'instances sont définies ici, rendant la configuration plus flexible et réutilisable.

outputs.tf :
Ce fichier définit les sorties du déploiement Terraform. Il inclut généralement des informations essentielles comme l'endpoint du cluster EKS et la configuration Kubernetes nécessaire pour interagir avec le cluster.

security_groups.tf :
Ce fichier gère les groupes de sécurité pour le cluster EKS. Il inclut les règles pour contrôler le trafic entrant et sortant vers et depuis les nœuds Kubernetes et le plan de contrôle.

eks_cluster.tf :
Ce fichier de configuration spécifique inclut les détails pour créer le cluster EKS, en spécifiant des paramètres comme la version de Kubernetes, la configuration du VPC et les contrôles d'accès basés sur les rôles pour le cluster.

node_groups.tf :
Ce fichier définit les groupes de nœuds pour le cluster EKS. Il inclut des détails sur les types d'instances, les politiques de mise à l'échelle et les configurations de bootstrap nécessaires pour que les nœuds rejoignent le cluster.

kubernetes.tf :
Ce fichier configure le provider Kubernetes dans Terraform, permettant le déploiement et la gestion des ressources Kubernetes au sein du cluster EKS. Il inclut souvent les configurations de déploiements et de services.

Ces fichiers, lorsqu'ils sont utilisés ensemble, facilitent le déploiement et la gestion d'un cluster EKS sur AWS en utilisant Terraform, garantissant une infrastructure évolutive et automatisée. Lien GitHub
